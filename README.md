# JWT Authentication API

This project demonstrates how to implement JWT authentication in a Spring Boot application, covering session management, generating a token upon user login, and securing routes.

## Features Let’s check off the core requirements:
- **Login Endpoint:** Authenticates users and returns a JWT token.
- **Protected Routes:** Certain URLs in the backend require a valid JWT token sent in the `Authorization: Bearer <token>` header.
- **Stateless Session:** Token validation is used securely.

## Prerequisites
- Java 17+
- Maven

## How to Run
1. Navigate to the project folder.
2. Build and run via Maven:
   ```bash
   ./mvnw spring-boot:run
   ```
3. The server runs at `http://localhost:8080`.

## Endpoints overview

### 1. `POST /login`
Accepts a JSON payload with credentials. 
Default defined user for testing:
- **username**: `user123`
- **password**: `password123`

### 2. `GET /protected`
Accessible only to users carrying a valid JWT Token.

## 🛠 Automated Terminal Testing (Alternative to Postman)

If you prefer to capture your tests in the terminal without opening Postman, an automated script is included.

Run the test script located in the root of the project. It will automatically:
1. Try to access the protected route without logging in (Fail).
2. Login and retrieve a valid JWT token (Success).
3. Access the protected route using that JWT token (Success).
4. Simulate a logout by providing an invalid/expired token (Fail).

**Command to run:**
```bash
./test_api.sh
```

You can simply take a screenshot of the terminal output generated by this script as proof of functionality!
